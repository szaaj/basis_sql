07

Subqueries - Queries in Queries

Gebruik subqueries voor complexe analyses

Voortgang 0/5 oefeningen

💡 Wat ga je leren?

Je hebt geleerd hoe je data filtert, groepeert en analyseert. Nu ga je leren hoe je queries binnen queries gebruikt voor complexe analyses!

Wat zijn Subqueries?

Een subquery is een query binnen een andere query. De subquery wordt eerst uitgevoerd, het resultaat wordt gebruikt in de hoofdquery.

Voorbeeld: Films duurder dan gemiddelde 
SELECT filmnaam, duur
FROM film
WHERE duur > (SELECT AVG(duur) FROM film);
💡 Subquery volgorde:

1. Binnenste query wordt eerst uitgevoerd (de subquery)

2. Resultaat wordt gebruikt in buitenste query (de hoofdquery)

Subqueries met IN

Gebruik IN als je subquery meerdere waarden teruggeeft:

Voorbeeld: Films in populaire genres 
SELECT filmnaam
FROM film
WHERE genreid IN (
    SELECT genreid 
    FROM film 
    GROUP BY genreid 
    HAVING COUNT(*) > 5
);

Subqueries in HAVING

Je kunt subqueries ook gebruiken in HAVING clausules:

Voorbeeld: Genres boven gemiddelde 
SELECT genreid, AVG(duur) AS gem_duur
FROM film
GROUP BY genreid
HAVING AVG(duur) > (SELECT AVG(duur) FROM film);

🎯 Oefeningen

Oefening 1 - Films langer dan gemiddelde

Gemiddeld

Vind alle films die langer zijn dan de gemiddelde filmduur. Toon filmnaam en duur. Gebruik een subquery met AVG().

SQL Query
💡 Subquery in WHERE met AVG
💡 Hint

WHERE duur > (SELECT AVG(duur) FROM film). "Langer" betekent > (niet >=).

✅ Oplossing 
SELECT filmnaam, duur
FROM film
WHERE duur > (SELECT AVG(duur) FROM film);

Oefening 2 - Regisseurs van populaire films

Gemiddeld

Vind alle regisseurs die minimaal 1 film hebben gemaakt van meer dan 150 minuten. Gebruik een subquery met IN. Toon alleen regisseur (uniek).

SQL Query
💡 IN subquery + DISTINCT
💡 Hint

Subquery: SELECT filmid FROM film WHERE duur > 150. "Meer dan 150" = > 150.

✅ Oplossing 
SELECT DISTINCT regisseur
FROM film
WHERE filmid IN (
    SELECT filmid 
    FROM film 
    WHERE duur > 150
);

Oefening 3 - Genres met lange gemiddelde duur

Moeilijk

Vind genres waar de gemiddelde duur hoger is dan de totale gemiddelde filmduur. Gebruik subquery in HAVING. Toon genreid en gemiddelde_duur.

SQL Query
💡 Subquery in HAVING
💡 Hint

GROUP BY genreid, HAVING AVG(duur) > (SELECT AVG(duur) FROM film). "Hoger" = >.

✅ Oplossing 
SELECT genreid, AVG(duur) AS gemiddelde_duur
FROM film
GROUP BY genreid
HAVING AVG(duur) > (SELECT AVG(duur) FROM film);

Oefening 4 - Top 3 langste films per land

Moeilijk

Vind de top 3 langste Amerikaanse films. Filter op land (WHERE), sorteer op duur (DESC), gebruik LIMIT. Toon filmnaam, regisseur en duur.

SQL Query
💡 WHERE + ORDER BY + LIMIT
💡 Hint

WHERE landherkomst = 'USA', ORDER BY duur DESC, LIMIT 3.

✅ Oplossing 
SELECT filmnaam, regisseur, duur
FROM film
WHERE landherkomst = 'USA'
ORDER BY duur DESC
LIMIT 3;

Oefening 5 - Regisseurs boven gemiddeld productief

Moeilijk

Vind regisseurs die meer films hebben gemaakt dan de gemiddelde regisseur. Gebruik subquery in HAVING. Toon regisseur en aantal_films.

SQL Query
💡 Nested subquery met AVG
💡 Hint

HAVING COUNT(*) > (SELECT AVG(aantal_films) FROM (SELECT COUNT(*) AS aantal_films FROM film GROUP BY regisseur)). "Meer" = >.

✅ Oplossing 
SELECT regisseur, COUNT(*) AS aantal_films
FROM film
GROUP BY regisseur
HAVING COUNT(*) > (
    SELECT AVG(aantal_films)
    FROM (
        SELECT COUNT(*) AS aantal_films
        FROM film
        GROUP BY regisseur
    )
);

🐛 Veelgemaakte fouten

❌ Subquery geeft meerdere waarden bij =

WHERE duur = (SELECT duur FROM film WHERE genreid = 1)

Als subquery meerdere waarden geeft, gebruik IN in plaats van =!

WHERE duur IN (SELECT duur FROM film WHERE genreid = 1)

❌ Alias niet beschikbaar in subquery

SELECT genreid, AVG(duur) AS gem
FROM film
GROUP BY genreid
HAVING gem > (SELECT AVG(duur) FROM film)  -- gem bestaat niet!

Gebruik de volledige functie in HAVING, niet de alias!

HAVING AVG(duur) > (SELECT AVG(duur) FROM film)
